# logo-project-workflow

> **Full Logo Project Workflow** - Research to sketches to refinement to system to delivery
> Complete logo design process following The Draplin Way methodology

## Workflow Definition

```yaml
workflow:
  name: logo-project-workflow
  agent: aaron-draplin
  category: logo-design
  type: sequential
  estimated_duration: "1-3 weeks depending on scope"

  description: |
    Complete logo project workflow from initial research through final delivery.
    Follows Aaron Draplin's iterative methodology: absorb deeply, sketch explosively,
    refine digitally, simplify ruthlessly, expand systematically, present confidently,
    deliver completely.

  phases:
    - research_discovery
    - sketch_exploration
    - digital_refinement
    - simplification_validation
    - system_creation
    - presentation_approval
    - final_delivery

  deliverables:
    - "Complete logo brief document"
    - "Concept exploration sketches"
    - "3-5 refined digital concepts"
    - "Final approved logo"
    - "Complete lockup system"
    - "All color variations"
    - "Usage guidelines"
    - "Organized file package"
```

---

## Draplin Principles Throughout

```
GUIDING PRINCIPLES FOR EVERY PHASE:

1. THICK LINES - Bold, confident strokes that work at any size
2. SIMPLIFY - "Remove until it hurts"
3. B&W FIRST - If it doesn't work in black and white, it doesn't work
4. COIN TEST - Must function at 16px/coin size
5. SKETCH FAST - Don't overthink, iterate quickly
6. VINTAGE-MODERN - Timeless structure, contemporary execution
7. WORK HARD - "Design is work, not magic"
8. MAKE COOL SHIT - Have fun, create something you're proud of
```

## Phase 1: Research & Discovery

```yaml
phase: research_discovery
duration: "2-4 hours"
gate: "Brief approved by client"
```

### 1.1 Initial Meeting/Intake

```markdown
## DISCOVERY OBJECTIVES

Understand deeply:
- [ ] What the business DOES
- [ ] Who the business SERVES
- [ ] What makes them DIFFERENT
- [ ] Where the logo will LIVE (applications)

### Questions to Ask

**CORE IDENTITY**
- "What does your business do in one sentence?"
- "What is the ONE WORD that captures your essence?"
- "Why was the business started?"
- "What makes you different from competitors?"

**AUDIENCE**
- "Describe your ideal customer."
- "Why do people choose you over alternatives?"
- "How do customers find you?"

**APPLICATIONS**
- "Where will this logo appear most often?"
- "Do you need it to work embroidered? On signs? In apps?"
- "What's the smallest it needs to work?"

**REFERENCES**
- "Show me 3 logos you love. Why do you love them?"
- "Show me 3 logos you hate. Why don't they work for you?"
- "Who are your competitors? What do their logos look like?"
```

### 1.2 Research Deep Dive

```markdown
## RESEARCH ACTIVITIES

### Competitor Analysis
- [ ] Identify 5-10 competitors
- [ ] Document their logo styles
- [ ] Note visual patterns in the industry
- [ ] Identify opportunities to differentiate

### Industry Visual Language
- [ ] What's expected in this space?
- [ ] What visual cues signal this industry?
- [ ] How can we meet expectations while standing out?

### Historical/Vintage Research (Draplin Style)
- [ ] Find vintage examples in related industries
- [ ] Gas station logos, old badges, heritage brands
- [ ] Extract structural principles, not copy styles

### Application Research
- [ ] How will logo be used most?
- [ ] What are the technical requirements?
- [ ] Size constraints, color limitations?
```

### 1.3 Brief Documentation

```markdown
## COMPLETE THE BRIEF

Use task: logo-design-process.md (Phase 1)

Brief Must Include:
- [ ] Essence word (ONE word)
- [ ] Business overview
- [ ] Target audience
- [ ] Personality spectrum
- [ ] Critical applications list
- [ ] Color requirements/restrictions
- [ ] Reference logos (love and hate)
- [ ] Competitors to avoid resembling
- [ ] Timeline and budget

### Brief Approval
- [ ] Present brief summary to client
- [ ] Confirm understanding is accurate
- [ ] Get written/verbal approval to proceed
```

### Phase 1 Gate

```
PROCEED TO PHASE 2 WHEN:
- [ ] Brief is complete
- [ ] Client has approved brief
- [ ] Essence word is defined
- [ ] Applications are clear
- [ ] Timeline is agreed
```

### 2.1 Sketch Session Preparation

```markdown
## PREP FOR SKETCHING

Materials Ready:
- [ ] Thick markers (Sharpie, felt tip)
- [ ] Stack of paper or sketchbook
- [ ] Brief summary visible
- [ ] Timer for pace

Mental Preparation:
- [ ] Review brief essence word
- [ ] Review references and anti-references
- [ ] Clear mind of assumptions
- [ ] Ready to explore without judgment
```

### 2.2 Sketch Explosion

```markdown
## THE SKETCH SESSION

"If you can't draw it in 10 seconds, it's too complicated."

### Round 1: Wordmark Explorations (15 min)
- [ ] All caps variations
- [ ] Mixed case variations
- [ ] Initial/monogram options
- [ ] Custom lettering ideas
Target: 10+ sketches

### Round 2: Icon/Symbol Explorations (15 min)
- [ ] Abstract geometric shapes
- [ ] Symbolic representations
- [ ] Letter-based marks
- [ ] Negative space concepts
Target: 10+ sketches

### Round 3: Combination Explorations (15 min)
- [ ] Icon + wordmark horizontal
- [ ] Icon + wordmark stacked
- [ ] Integrated marks
- [ ] Badge/emblem structures
Target: 10+ sketches

### Round 4: Wild Card (10 min)
- [ ] Unexpected directions
- [ ] Break the rules
- [ ] What if we tried...?
Target: 5+ sketches

TOTAL TARGET: 35-50 thumbnail sketches
```


## Phase 3: Digital Refinement

```yaml
phase: digital_refinement
duration: "4-8 hours"
gate: "3 concepts ready for presentation"
```

### 3.1 Vector Construction

```markdown
## BUILD IN VECTOR

For each selected direction:

### Construction Process
1. [ ] Create new artboard (standard size: 500x500px)
2. [ ] Build using geometric shapes
3. [ ] Apply thick lines philosophy
4. [ ] Work in BLACK ONLY (no color yet)
5. [ ] Refine proportions
6. [ ] Clean up paths

### Quality Standards
- [ ] No unnecessary anchor points
- [ ] Smooth curves, no jagged edges
- [ ] Stroke weight minimum 0.5pt at card size
- [ ] Consistent weight throughout
- [ ] Grid-aligned where appropriate

### Per Concept Checklist
Concept A:
- [ ] Vector built
- [ ] Paths clean
- [ ] Thick lines applied
- [ ] Proportions refined

[Repeat for B, C, D, E]
```

### 3.2 Initial Testing

```markdown
## EARLY TESTING

For each concept, before going further:

### Coin Test (Quick)
- [ ] Display at 16x16px
- [ ] Recognizable? Y/N
- [ ] If N, simplify now

### B&W Check
- [ ] Is it working in black?
- [ ] No dependency on color?
- [ ] If issues, address now

### Quick Squint Test
- [ ] Squint at arm's length
- [ ] Shape clear?
- [ ] Distinctive silhouette?
```

### 3.3 Refinement Iterations

```markdown
## REFINE EACH CONCEPT

### Iteration Loop
For each concept, cycle through:

1. Build version
2. Test at small size
3. Identify weak elements
4. Refine or remove
5. Test again
6. Repeat until solid

### Document Iterations
Keep versions to show process:
- Concept_A_v1.ai
- Concept_A_v2.ai
- Concept_A_v3.ai
- etc.

### When to Stop
A concept is "refined" when:
- [ ] Passes coin test
- [ ] Works in B&W
- [ ] Every element is necessary
- [ ] Further changes are lateral, not improvements
```

### Phase 3 Gate

```
PROCEED TO PHASE 4 WHEN:
- [ ] 3-5 concepts digitally refined
- [ ] All pass basic coin test
- [ ] All work in B&W
- [ ] Each is distinctly different
- [ ] Ready for deep simplification
```

```markdown
- Logo on dark

- Logo on dark
- Impactful mockup

- Refinement timeline
- What happens next
```
