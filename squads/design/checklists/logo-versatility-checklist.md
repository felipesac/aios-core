# logo-versatility-checklist

> **Logo Versatility Validation Checklist** - Ensure logo works across all applications
> Based on Aaron Draplin's Logo Versatility System framework

## Checklist Definition

```yaml
checklist:
  name: logo-versatility-checklist
  agent: aaron-draplin
  category: logo-design
  usage: "Run when creating logo system and before final delivery"

  purpose: |
    Validate that a logo system is complete and versatile enough to work
    in every possible application. "One logo, infinite applications."

  when_to_use:
    - After creating lockup system
    - Before final client delivery
    - When auditing existing logo system
    - When expanding logo applications
```

---

## Section 1: Lockup System Completeness

> "If you only deliver one version, you're not done."

### 1.1 Primary Lockup

```markdown
## PRIMARY LOCKUP

The main, preferred version of the logo.

Existence & Quality:
- [ ] Primary lockup exists
- [ ] Proportions finalized and documented
- [ ] Hierarchy clear (what reads first, second)
- [ ] Spacing between elements defined
- [ ] Master file created

Documentation:
- [ ] Aspect ratio documented
- [ ] Recommended use cases listed
- [ ] Minimum size defined
- [ ] File named correctly
```

### 1.2 Horizontal Lockup

```markdown
## HORIZONTAL LOCKUP

Icon + wordmark side by side.

Existence & Quality:
- [ ] Horizontal lockup exists
- [ ] Icon positioned correctly (left or right)
- [ ] Vertical alignment defined (center, baseline, etc.)
- [ ] Proportional spacing set
- [ ] Balance between icon and text

Specifications:
- [ ] Aspect ratio: ___:___ (typically 3:1 to 5:1)
- [ ] Icon to text spacing defined
- [ ] Works in headers and banners
- [ ] Minimum width: ___px / ___"

Use Cases Covered:
- [ ] Email signatures
- [ ] Website headers
- [ ] Document headers
- [ ] Social media banners
- [ ] Vehicle graphics (side view)
```

### 1.3 Stacked Lockup

```markdown
## STACKED LOCKUP

Icon above wordmark (vertical arrangement).

Existence & Quality:
- [ ] Stacked lockup exists
- [ ] Icon centered above text
- [ ] Vertical spacing defined
- [ ] Visual weight balanced
- [ ] Works in square contexts

Specifications:
- [ ] Aspect ratio: ___:___ (typically 1:1 to 1:1.5)
- [ ] Icon to text spacing defined
- [ ] Works for square placements
- [ ] Minimum height: ___px / ___"

Use Cases Covered:
- [ ] Social media avatars
- [ ] Business card centered layout
- [ ] Promotional squares
- [ ] App store listing
- [ ] Signage (vertical)
```

### 1.4 Icon Only

```markdown
## ICON ONLY (MARK)

Symbol/mark without any text.

Existence & Quality:
- [ ] Icon-only version exists
- [ ] Works completely independently
- [ ] Recognizable as the brand
- [ ] Passes coin test alone
- [ ] Has enough presence without text

Specifications:
- [ ] Aspect ratio: 1:1 (or natural proportion)
- [ ] Minimum size: 16px (favicon requirement)
- [ ] Optimized versions for different sizes exist

Size Optimizations:
- [ ] 16px version (may need simplification for favicon)
- [ ] 32px version
- [ ] 64px version (app icon standard)
- [ ] 128px+ version (full detail)
- [ ] 512px+ version (app store)

Use Cases Covered:
- [ ] Favicon
- [ ] App icons (iOS, Android)
- [ ] Social media profile pictures
- [ ] Watermarks
- [ ] Very small placements
- [ ] Pattern/texture use
```

### 1.5 Wordmark Only (Optional)

```markdown
## WORDMARK ONLY

Text without icon (if applicable to brand).

Existence Assessment:
- [ ] Wordmark-only version needed? [ ] Yes [ ] No
- [ ] If yes, wordmark-only version exists
- [ ] Maintains brand typography
- [ ] Same proportions as in lockups

Use Cases (if applicable):
- [ ] Text-heavy contexts
- [ ] Legal documents
- [ ] When icon is redundant
- [ ] Simplified brand mentions

Note: Not all brands need wordmark-only. Assess need.
```

### 1.6 Badge Version

```markdown
## BADGE VERSION

Logo in emblem/badge structure (for merchandise, patches, etc.).

Existence & Quality:
- [ ] Badge version exists
- [ ] Container shape selected (circle, shield, etc.)
- [ ] Elements balanced within container
- [ ] Text on path legible (if used)
- [ ] Embroidery/patch viable

Specifications:
- [ ] Container shape: _________________
- [ ] Suitable for merchandise
- [ ] Maximum 6 colors for embroidery
- [ ] Works at patch sizes (1" to 4")

Use Cases Covered:
- [ ] Embroidered patches
- [ ] Enamel pins
- [ ] Merchandise (hats, shirts)
- [ ] Certificates/credentials
- [ ] Awards/recognition
- [ ] Formal applications
```

## Section 2: Color Variations

> "Black and white first. Color is dessert."

### 2.1 Full Color

```markdown
## FULL COLOR VERSIONS

Complete branded color version.

RGB Version (Digital):
- [ ] RGB values documented
- [ ] HEX codes documented
- [ ] sRGB color profile applied
- [ ] File saved as RGB

CMYK Version (Print):
- [ ] CMYK values documented
- [ ] Conversion accurate
- [ ] Print-ready file exists
- [ ] Rich black formula noted (if used)

Pantone Version (Spot):
- [ ] Pantone code(s) documented
- [ ] Coated version: _________________
- [ ] Uncoated version: _________________
- [ ] Spot color file exists
```

### 2.2 One Color Versions

```markdown
## ONE COLOR VERSIONS

For limited color applications.

Black Version (100% K):
- [ ] Pure black version exists
- [ ] Works on white/light backgrounds
- [ ] All elements visible
- [ ] File saved correctly

Use cases:
- Newsprint
- Fax (yes, still exists)
- Rubber stamps
- Engraving
- Budget printing

White Version (100% White):
- [ ] Pure white version exists
- [ ] Works on dark backgrounds
- [ ] Optical adjustments made if needed
- [ ] File saved with transparency

Use cases:
- Dark backgrounds
- Screen printing on dark fabrics
- Foil stamping
- Window decals

Primary Brand Color Version:
- [ ] Single brand color version exists
- [ ] Uses primary brand color
- [ ] Works for promotional items

Use cases:
- Promotional products
- Single-color merchandise
- Sponsor placements
```

### 2.3 Reversed Versions

```markdown
## REVERSED VERSIONS

For dark backgrounds.

Reversed Assessment:
- [ ] All lockups have reversed versions
- [ ] White/light version for dark backgrounds
- [ ] Tested on brand dark colors
- [ ] Tested on black
- [ ] Tested on photo backgrounds

Optical Adjustments:
Does the logo need optical adjustment when reversed?
- [ ] No adjustment needed (rare)
- [ ] Stroke weight increased slightly
- [ ] Spacing adjusted
- [ ] Separate reversed file created

Common Issues Checked:
- [ ] No elements disappear
- [ ] Contrast sufficient
- [ ] Details remain visible
- [ ] Overall balance maintained
```

### 2.4 Grayscale

```markdown
## GRAYSCALE VERSION

For black and white printing with tones.

Grayscale Version:
- [ ] Grayscale version exists
- [ ] Colors converted to appropriate gray values
- [ ] Hierarchy maintained
- [ ] Contrast sufficient

Use cases:
- Newspaper advertising
- Black and white documents
- Photocopies
- Some packaging
```
